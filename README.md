# 游戏积分计算器

一个纯前端的游戏积分结算工具，适合多人线下对局后快速核对输赢、抽水和转账方案。

页面使用原生 `HTML + CSS + JavaScript` 编写，不依赖后端服务，也不需要打包构建，直接打开即可使用，部署到 GitHub Pages 也很方便。

## 功能简介

- 支持动态添加和删除玩家
- 录入每位玩家的取分和剩余积分
- 支持设置抽水/场地费
- 支持指定场地费收款人
- 自动计算每位玩家输赢结果
- 自动校验数据是否平衡
- 自动生成结算明细和简化转账方案
- 支持生成测试数据，方便演示和自测
- 使用 `localStorage` 自动保存当前页面数据
- 兼容手机和桌面端浏览器

## 文件结构

当前项目目录如下：

```text
integral/
├─ index.html
├─ integral.html
├─ .nojekyll
└─ README.md
```

文件说明：

- `index.html`：入口页
- `integral.html`：积分计算器主页面
- `.nojekyll`：用于 GitHub Pages，避免静态资源被 Jekyll 处理
- `README.md`：项目说明文件

## 本地使用

### 直接打开

直接在浏览器中打开以下任一文件即可：

- `index.html`
- `integral.html`

更推荐从 `index.html` 进入。

### 使用流程

1. 输入抽水总额
2. 选择场地费由谁统一收取
3. 为每位玩家填写姓名、取分和剩余积分
4. 页面会自动计算输赢结果
5. 查看结算统计、校验状态和转账方案

## 数据保存说明

页面会使用浏览器的 `localStorage` 自动保存当前输入的数据。

这意味着：

- 刷新当前页面后，数据通常会保留
- 同一台设备上的同一浏览器可以继续使用上次数据
- 不同设备之间不会自动同步数据
- 更换浏览器或清除浏览器缓存后，数据可能丢失

## GitHub Pages 部署

这个项目是纯静态页面，适合直接部署到 GitHub Pages。

### 方式一：直接从分支发布

如果你的仓库内容就是当前这些文件，推荐使用这个方式。

1. 将项目推送到 GitHub 仓库
2. 打开仓库页面，进入 `Settings`
3. 点击左侧 `Pages`
4. 在 `Build and deployment` 中选择：
5. `Source`：`Deploy from a branch`
6. `Branch`：`main`
7. `Folder`：`/ (root)`
8. 保存设置，等待 GitHub 部署完成

部署完成后，通常可以通过下面的地址访问：

```text
https://你的用户名.github.io/仓库名/
```

### 方式二：作为仓库子目录发布

如果这个项目将来会放在一个更大的仓库里，只作为其中一个子目录使用，那么建议额外配置 GitHub Actions，把该子目录单独发布到 Pages。

## 技术特点
